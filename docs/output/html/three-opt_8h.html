This functions does a 3-opt swap of three edges of a path.

Graph  g

This functions improves a path using the 3-opt algorithm, restricted to the edges of the graph.

This function improves a path using the 3-opt algorithm. This Algorithm starts with a given path and tries to improve it by doing a single execution of 3-opt. For every combination of three different edges the algorithm verifies if one of the 7 possible swap improves the path, it is done, otherwise if continues to the next combination of 3 edges. It first calls the function verify3Opt to check if the swap improves the path and which is the best option, and then it calls the function do3Opt to do the swap. Before running the 3-opt algorithm, it runs the 2-opt algorithm to get better results.

This function returns a path restricted by the edges of the graph

The return of this function is not guaranteed to be the best path possible, it is just an heuristic

Time Complexity: O(V^4) where V is the number of vertices in the graph, that is the same as the number of edges in the path

Parameters
pathvector of edges that represent the path
ggraph that contains the edges of the path and that restrictes the edges that can be added
Returns
an improved path restricted by the edges of the graph

Calculates which of the 7 possible 3-opt swaps is the best.

This function sums the distance of the 3 edges that are going to be removed and then compares the sum with the sum of the 3 edges that could be added after that returns the number that represents the option that has the lowest sum If the best option is to not do the swap returns 0

Time Complexity: O(1)

iindex of the first edge
